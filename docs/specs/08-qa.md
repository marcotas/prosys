# QA — End-to-End Verification

- **Type**: qa
- **Depends on**: integration
- **Files owned**: None (read-only pass)
- **Interfaces exposed**: Verified, working application
- **Interfaces consumed**: Everything from all tasks + integration

## Description

Systematic end-to-end verification of all features, integration points, and edge cases. Run the dev server and test every acceptance criterion using a browser. Open two tabs side-by-side to verify real-time sync.

## Test Checklist

### 1. Member CRUD

- [ ] App loads with seed members (4 default profiles)
- [ ] Create a new member via "+" button → profile dialog opens
- [ ] Fill name, pick theme, pick emoji → save → member appears in switcher
- [ ] Edit an existing member → changes persist after page reload
- [ ] Delete a member → removed from switcher, another member auto-selected
- [ ] Switch between members → tasks and habits refresh for selected member

### 2. Task CRUD

- [ ] Add a task via the input field on any day card → task appears immediately
- [ ] Add a task with emoji → emoji displays next to title
- [ ] Toggle task completion (checkbox) → visual state updates, progress ring updates
- [ ] Edit task title inline (tap/click title) → changes persist
- [ ] Delete task via swipe-reveal menu → task removed, progress updates
- [ ] Empty day card shows placeholder / add prompt

### 3. Habit Tracking

- [ ] Add a habit via the input field → habit row appears in the tracker
- [ ] Add a habit with emoji → emoji displays next to name
- [ ] Toggle habit completion for a specific day → checkbox fills, progress bar updates
- [ ] Edit habit name inline → changes persist
- [ ] Delete habit via swipe-reveal menu → habit removed
- [ ] Empty habit tracker shows placeholder / add prompt

### 4. Drag & Drop — Tasks

- [ ] Drag-reorder tasks within a day (grab handle, drag up/down) → order persists after reload
- [ ] Drag a task from one day card to another (desktop) → task moves, both days update
- [ ] "Move to..." via swipe-reveal menu (mobile) → day picker appears, task moves to selected day
- [ ] After cross-day move, task lands at correct position in target day
- [ ] Drag shadow/placeholder shows during drag

### 5. Drag & Drop — Habits

- [ ] Drag-reorder habits (grab handle, drag up/down) → order persists after reload
- [ ] Drag shadow/placeholder shows during drag

### 6. Week Navigation

- [ ] "Previous week" button → loads tasks and habits for prior week
- [ ] "Next week" button → loads tasks and habits for next week
- [ ] "Today" button → returns to current week
- [ ] Week label updates to show correct date range
- [ ] Tasks added in a past/future week persist when navigating back

### 7. Real-Time Sync (two-tab test)

Open two browser tabs to the same URL side-by-side:

- [ ] Create a member in Tab A → appears in Tab B switcher
- [ ] Update a member in Tab A → name/theme updates in Tab B
- [ ] Delete a member in Tab A → removed from Tab B
- [ ] Add a task in Tab A → appears in Tab B on correct day
- [ ] Toggle a task in Tab A → completion state updates in Tab B
- [ ] Delete a task in Tab A → removed from Tab B
- [ ] Reorder tasks in Tab A → new order reflected in Tab B
- [ ] Move task to different day in Tab A → task moves in Tab B
- [ ] Toggle a habit in Tab A → checkbox updates in Tab B
- [ ] Add a habit in Tab A → appears in Tab B
- [ ] Delete a habit in Tab A → removed from Tab B
- [ ] Reorder habits in Tab A → new order reflected in Tab B
- [ ] Connection indicator shows green dot when connected

### 8. Progress & Stats

- [ ] Overall progress ring shows correct completion percentage
- [ ] Weekly bar chart shows per-day completion counts
- [ ] Progress updates in real-time as tasks are toggled
- [ ] Habit progress bars show correct fill per habit

### 9. Error States & Edge Cases

- [ ] Empty state when no members exist → "Create First Profile" prompt
- [ ] Long task titles wrap correctly (no overflow)
- [ ] Long habit names wrap correctly
- [ ] Rapidly toggling a task does not cause race conditions
- [ ] Creating many tasks in quick succession works correctly
- [ ] No console errors in Chrome DevTools during normal operation

### 10. Visual & Responsive

- [ ] Desktop layout: 7-column day grid
- [ ] Tablet layout: day cards reflow to fewer columns
- [ ] Mobile layout: single-column stack
- [ ] Theme colors apply correctly (accent, header, progress ring)
- [ ] Playful vs. default theme variant renders correctly
- [ ] Transitions/animations are smooth (no jank)
- [ ] Swipe gestures work on touch devices / touch simulation

## Acceptance Criteria

All checklist items pass. Zero console errors during testing. Real-time sync works reliably across tabs.
